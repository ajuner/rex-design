### 设计目标

- 使得**企业级中后台设计相关**的 Designer 能够**快速了解**自己的**问题或需求是否可以在这里得到解决**。
- 提供**明确的说明、资料工具、协作机制和路径**，帮助设计师**做出符合规范标准的设计**。
- 通过务实、负责且开放的态度，**推进**后台系统设计规范的**不断完善和优化**。

### 设计内容

- **框架**：系统平台框架/ 角色层工作台框架/应用层页面框架
- **模版**：通用模版 / 可视化模版 / 业务特色模版
- **组件**：基础组件 / 可视化组件 /
- **元素**：色彩 / 文字 / 间距 / 圆角 / 阴影 / 图标 / 动效 等

另外，还包含提升设计和开发效率的**设计资源包。**

### 设计策略

#### 原则

Designer 要根据规范物料颗粒度的不同，区分组件规范和模版规范的”态度差异“。

组件规范—— 是 **「基础原则」和「底线」。**

模版规范—— 更多的是 **「最佳实践」与「建议」。**

#### 策略

在设计上根据业务特性遵循克制的原则。

- 抛开具体业务线，从设计上抽象标准通用和稳定的后台规范；
- 从多条业务线抽象出复用性强的设计规范；
- 求同存异，在 2 的基础上，允许各个业务线保留自己业务定制的模块/模版。

#### 执行

设计规范的沉淀是需要 Designer 要在设计过程中有完成的推导链路。保证使用者在规范应用的使用场景是准确的。

另外，我们会把 Designer 在规范使用中的**高频应用问题反馈**的一些组件，进行详细的规范文档撰写，帮助各业务设计师**更快更准确**的进行设计判断和场景使用。

### 结语

我们不强调设计资源库是一种生搬硬套，去框死设计师的设计创造驱动力。**而是一种指引，一种设计的最佳实践。我们大家一起共建和互通，让设计规范越来越丰富，体验越来越好。**

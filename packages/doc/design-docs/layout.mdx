# 栅格系统

**Breakpoint 规则**

**![编组 2备份@2x.png#width=800px](https://hippo-oss.oss-cn-hangzhou.aliyuncs.com/rex/design-images/layout/89b7275e.png)**

我们需要一种弹性的栅格系统，它遵循如下原则：

1.  由于新零售 b 类场景特点是信息密度高，考虑到屏幕利用率和可读性，采用“紧凑型”栅格布局规则

2.  布局以 4 栅格作为响应式断点的最小依据

![image.png#width=600px](https://hippo-oss.oss-cn-hangzhou.aliyuncs.com/rex/design-images/layout/168a7702.png)

![编组 6@3x.png#width=400px](https://hippo-oss.oss-cn-hangzhou.aliyuncs.com/rex/design-images/layout/d6243f55.png)

注意：响应式栅格的列宽是随着页面宽度变化的，因此用百分比来表示

我们设定：以 4 为栅格；水槽固定宽为 12px；最小列宽为 80px

# **关于【4】**

分辨率与最小栅格单位在 x、y 轴整除关系表。「 4 」是整除率最高的一个原子

![image.png#width=800px](https://hippo-oss.oss-cn-hangzhou.aliyuncs.com/rex/design-images/layout/e3d57c66.png)

# 偶数原则

在页面缩放中最大程度的避免类似于 0.5、0.75、1.25 等次像素的出现，从而使页面各类元素在大多数场景下都能有比较精致的细节表现。

![image.png#width=424.5px](https://hippo-oss.oss-cn-hangzhou.aliyuncs.com/rex/design-images/layout/009a8e72.png)

# 列和槽

按容器尺寸划分响应式断点的规则如下：

· \[360, 764)：  列数 4； 水槽 12px ; （如有外边距，w=20px ）.

· \[764, 1132)： 列数 8；  水槽 12px ; （如有外边距，最小为 w=20px ）.

· \[1132, 1500)：列数 12； 水槽 12px ; （如有外边距，最小为 w=20px ）.

· \[1500, \*)：    列数 16； 水槽 12px ; （如有外边距，最小为 w=20px ）.

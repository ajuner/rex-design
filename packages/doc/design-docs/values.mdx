ReX Design OS 孵化于行业领先的生鲜零售 O2O 变革者，立足于更高效地推进生鲜行业 Farm2Table 的全链条数智化升级改造，提供给消费者更为确定性更高品质的生鲜消费体验；为了实现这一点，首先需要最大地释放出生鲜供应链及零售企业内的经营及生产活动的效能。

在零售企业内部，从总部 区域 门店 档口 的组织树关系，到基于事务的跨职能横向协同，涉及角色复杂，设备端的应用也越来越丰富，端与端之间的联动也愈加频繁。在设计系统的建设上，过去基于具体设备端的构造方式已经无法满足快速变化的零售场景诉求。

我们重新去思考影响数字界面和设计系统的因素有哪些，在近 5 年的零售操作系统演进中，我们核心识别出的影响因子可以归类为人，机，法，环四个纬度；

### 人因 Human Factor

界面是用户与数字世界交互的介质，设计系统需要回归到界面的用户本身。最基本的交互距离，操作模态，以及对于用户视弱，色盲，重听等障碍因素，都会影响到界面的表达。

### 机因 Device Factor

设备是设计系统通常考量的最关键因素，从设备的分辨率，输入形式（键鼠，触控，语音，手势等）和输出信道（屏幕，指示灯，铃音，震动等），到设备形态（握持，穿戴，台式，壁挂等），都涉及到相应的响应和适配规则。

### 法因 Task Factor

进入到具体的业务场景设计，我们会区分出信息的高低密度，操作的移动和静止，容错性和难易度的高低，从而匹配不同的设计策略；

### 环因 Environment Factor

环境对界面的影响是渐进式的，比如环境光线的强弱变化，还有环境噪音和温湿度的高低，都会被纳入到对界面设计的考量中。

基于以上对于人，机，法，环的因子拆解，我们进一步去探索，这些因子中哪些可被数字刻画，以及每个因子分别以怎样的方式对于我们的数字界面产生影响，这就是 ReX Design OS ：Multi-Factor Design Token 的开始；

在多因子拆解的同时，我们也对于这套设计系统应用场景做了愿景定义；

因为源于不断数字化升级的零售行业 B 类场景，我们定义了我们的设计价值观为 **角色感 真实感 连接感 生长性。**

### 角色感

在过往的零售行业，职业化的角色感来自于制服和工具，在数字化的时代，职业的角色感更多来自于每日工作高频使用的数字化作业界面。

- 数字化的工作界面本身岗位角色心智的表达；
- 系统记录和习得角色的使用习惯，提供角色化定制能力；
- 系统的设计可反向塑造角色的行为和习惯；

### 真实感

零售的数字演进是对于人-货-场的重构，重构的基础是客观真实世界人-货-场的数字化孪生刻画；

- 数字系统需能客观实反应真实世界，可建模可追溯；
- 数字系统的交互，可传导至真实世界并产生影响；
- 真实世界与数字系统的保持数据的同步；

### 连接感

在零售企业的数字化经营中，99%的作业场景都需要协同完成。在界面中需表达出这种协同的连接关系；

- 协同的连接可以是人与人，也可以是人与硬件设备；
- 协同的系统性需完成人与设备的信息数字化，包括其状态信息；系统提供全局的关系图谱；
- 连接可显性化表达的，亦可以无感。

### 生长性

不管是角色还是系统还是设备，都不断的进化和生长，设计系统应该支撑该生长性；
